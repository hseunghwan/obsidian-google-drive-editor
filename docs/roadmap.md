# 기능 로드맵 — "실제 Obsidian을 쓰는 경험"을 향해 (2026-07-07)

72개 기능 후보를 발굴하고(도메인 6개 + 누락 보완 12개), 세 가지 렌즈로 전수 채점해 우선순위를 정한 문서.

- **uv (사용자 가치 ×2 가중)** — Obsidian 사용자가 이 확장에서 실제로 아쉬워할 순서. 매일 쓰는 기능 우선.
- **rk (구현 리스크)** — 현 아키텍처(CM6 라이브 프리뷰, 원문 보존, Drive API, Chrome 확장)에 자연스럽게 얹히면 높음.
- **pf (철학 적합성)** — "기존 파일 그대로 / 작성 흐름 방해 금지 / 저장 안전 우선 / Obsidian 대체가 아닌 가벼운 편집 공간".

총점 = uv×2 + rk + pf (최대 20). 규모: S=하루 이내, M=며칠, L=몇 주.

관련 문서: [obsidian-command-palette-review.md](obsidian-command-palette-review.md) (팔레트 97개 전수 판정 — 후순위 20개가 본 로드맵에 흡수됨).

---

## Phase 1 — 작성 흐름 완성 (지금 바로, 대부분 S)

한 번의 작성 세션에서 손에 걸리는 마찰 제거. 전부 기존 스택 배선 수준.

| 기능 | 점수 | 규모 | 요지 |
|---|---|---|---|
| 자동 목록 이어가기 | 20 | S | Enter 시 `- ` / `1. ` / `- [ ] ` 자동 이어가기, 빈 항목에서 해제. 이미 설치된 `@codemirror/lang-markdown`의 `insertNewlineContinueMarkup` 배선만. 최다 체감 기능 |
| 위키링크·링크 클릭 이동 | 19 | M | 라이브 프리뷰의 `[[위키링크]]` 클릭 → `VaultIndex`로 해당 노트 열기, 외부 URL은 새 탭. 노트 간 이동이 없으면 vault가 아니라 파일 뭉치다. 뒤로/앞으로(Phase 2)의 전제 |
| 토큰 자동 갱신·429 백오프 | 18 | S | `chrome.identity` 토큰 만료 시 재발급, Drive 429/5xx 지수 백오프. 장시간 세션에서 저장 실패를 원천 감소 — 철학(저장 안전) 정중앙 |
| 문서 내 찾기/바꾸기 (⌘F) | 17 | S | `@codemirror/search` 배선 + 다크 테마 CSS. 탭 페이지라 ⌘F 가로채기 가능 |
| 세션 복원 | 17 | S | 마지막 활성 문서·커서 위치·스크롤을 localStorage에 저장, 재시작 시 이어서. "열던 곳에서 계속"은 Obsidian의 기본 감각 |
| 헤딩·목록 접기 | 14 | S | lang-markdown의 foldService 기반 `codeFolding()`+`foldKeymap` 배선. 팔레트 후순위 3건(#36/#37/#85) 일괄 해소 |

## Phase 2 — Obsidian다운 탐색·렌더링 (단기)

"내 vault를 돌아다니는" 경험과 라이브 프리뷰 완성도.

| 기능 | 점수 | 규모 | 요지 |

| 뒤로/앞으로 문서 히스토리 | 16 | M | 문서 이동 스택 + ⌘⌥←/→. 위키링크 클릭 이동과 세트 (팔레트 #26/#71) |
| 포커스 모드 + 읽기 좋은 줄 길이 | 16 | S | 에디터 최대 폭 옵션 + 패널 숨김 토글. CSS 위주 |
| ==하이라이트== 렌더링 | 15 | S | 위키링크와 동일한 정규식 데코 패턴 하나 추가 |
| 검색·자동완성 frecency 랭킹 | 15 | S | 최근·자주 연 파일을 검색/자동완성 상위로 |
| 콜아웃 렌더링 | 14 | M | `> [!note]` 타입별 색·아이콘. 슬래시로 삽입은 되는데 렌더가 없어 반쪽 |

## Phase 3 — vault 운영 (중기)

매일 쓰는 도구가 되기 위한 운영 기능. Drive API 확장 필요.

| 기능 | 점수 | 규모 | 요지 |
|---|---|---|---|
| 템플릿 삽입 (변수 지원) | 16 | M | vault 내 `templates/` 폴더 + `{{date}}`/`{{title}}` 변수. 일일 노트의 전제 |
| 일일 노트 | 16 | M | 오늘 날짜 노트 열기/생성(경로·포맷 설정). Obsidian 최다 사용 워크플로 중 하나 |
| 본문 전문 검색 | 15 | M | Drive `fullText contains` 검색을 사이드바 검색에 통합(이름 검색과 결과 병합) |
| 폴더 간 파일 이동 | 14 | M | 사이드바 메뉴 "이동…" + 폴더 선택 다이얼로그(Drive parents PATCH). 팔레트 #21 |
| 이미지 표시 1단계: 외부 URL | 14 | M | `![](https://…)`를 테이블과 같은 StateField block widget으로 렌더 |
| 이미지 2단계: 붙여넣기 → Drive 업로드 | 14 | M | 클립보드 이미지를 vault 하위 `attachments/`에 업로드 후 링크 삽입 |
| 미저장 초안 목록 UI | 14 | S | IndexedDB에 보존된 초안들을 설정/사이드바에서 열람·복원·폐기. 이미 쌓고 있는데 보는 창구가 없음 |
| 충돌 해결 diff·병합 다이얼로그 | 14 | L | 원격 변경 감지 시 로컬/원격 diff 나란히 보여주고 선택 병합. 철학 적합성 만점(pf5) — 현재는 경고 아이콘뿐 |
| 실패 저장 자동 재전송 | 13 | M | 네트워크 복구 감지 시 초안 자동 재시도 |
| 원격 변경 감시(활성 파일 폴링) | 13 | S | 열려 있는 파일의 modifiedTime 주기 확인 → 저장 전에 미리 충돌 경고 |
| Drive 버전 히스토리 열람·복원 | 12 | M | Drive revisions API — Obsidian에 없는 Drive만의 강점 |
| 이미지 3단계: `![[image.png]]` Drive 임베드 | 13 | L | 비-md 파일 인덱싱 + 인증 다운로드 + blob 캐시. 1·2단계 이후 |

## Phase 4 — 지식 관리 확장 (장기·조건부)

가벼움을 해치지 않는 선에서. 착수 전 재평가.

| 기능 | 점수 | 규모 | 조건 |
|---|---|---|---|
| 백링크 패널 | 12 | L | 전문 검색(또는 백그라운드 인덱스) 선행 필요. README 비범위를 푸는 결정이라 명시적 재검토 필요 |
| 링크 호버 미리보기 | 12 | M | 문서 캐시 재사용. 백링크보다 싸고 체감 좋음 — 먼저 검토 |

| 노트 임베드 `![[노트]]` 렌더링 | 12 | M | 트랜스클루전. 호버 미리보기 이후 |

## 하지 않음 (재확인 전까지)

| 항목 | 이유 |
|---|---|
| 파일명 변경 시 위키링크 자동 갱신 | rk1·pf1 — 여러 파일 원문을 일괄 수정해야 해서 "기존 파일 그대로" 원칙과 충돌, 부분 실패 시 vault 오염 위험. 대안: 이름 변경 시 영향받는 링크 개수만 경고 |
| 수식(LaTeX)·Mermaid 렌더링 | 무거운 의존성 대비 사용 빈도 낮음. 요청 쌓이면 재검토 |
| 사용자 단축키 설정·2-pane 분할·vault 영속 인덱스 | 유지보수 부담 대비 가치 낮음 (uv≤3, rk≤2) |
| 모바일/PWA | 기능이 아니라 별도 제품 트랙. Chrome 확장 정체성 결정이 선행 |

## 요약 실행 순서

```
Phase 1 (S 6개): 목록 이어가기 → 링크 클릭 이동 → 토큰 갱신 → ⌘F → 세션 복원 → 접기
Phase 2: 빠른 전환기 → 히스토리 → 렌더링 마감(하이라이트/콜아웃/코드 하이라이팅) → 미해결 링크
Phase 3: 템플릿+일일 노트 → 전문 검색 → 파일 이동 → 이미지 1·2단계 → 충돌 병합 → 버전 히스토리
Phase 4: 재평가 후 착수
```

그래프 뷰(전체)는 2026-07-09 재평가로 착수 — `docs/superpowers/specs/2026-07-09-graph-view-design.md`
